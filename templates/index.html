What this app does

Learn preference weights from your store data

The pipeline follows the research direction you selected: first a decision tree predicts an engagement‑style target from simple customer and product fields; then a genetic algorithm (GA) learns how strongly each engineered signal (including the tree output) should contribute to a single ranking score. Those learned weights are your interpretable preference profile for the report, alongside tables and charts below.

  1. 1 Load the official HW__Data_S25 Excel bundle (users, products, ratings, behaviour).
  2. 2 Split rows into train / validation; build train‑only aggregates so validation scores stay honest.
  3. 3 Train the tree, run the GA on validation error, then inspect metrics, chart, and tree rules.
  4. 4 Open any user to see the same global weights applied to a personalised catalogue ranking (prices in €).
